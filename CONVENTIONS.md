# CONVENTIONS — 모든 세션 공통 규칙 (단일 출처)

> 이 문서는 `AI_COP` 레포의 **모든 세션(session2, session3, …)에 공통으로 적용되는 불변 규칙의 최신·단일 출처**입니다.
> `docs/` 의 세션별 스펙 문서는 참고용/이력이며, 범용 규칙이 충돌하면 **이 문서가 기준**입니다.

---

## 1. 레포 구조 (멀티세션 컨테이너)

이 레포는 박사후연구원 학습공동체의 회차(세션)별 실습 자료를 담는 컨테이너입니다.

```
AI_COP/
├─ README.md          # 레포 인덱스 (세션 목록/링크)
├─ CONVENTIONS.md     # 이 문서 (공통 규칙 단일 출처)
├─ .gitignore
├─ docs/              # 기획·스펙 문서 보관(참고용/이력)
└─ session2/          # 한 세션 = 자기완결 폴더
   ├─ README.md          # 세션 소개 + Open in Colab 배지
   ├─ requirements.txt   # 세션별 의존성(버전 고정)
   ├─ setup_env.bat / setup_env.sh   # 로컬 conda 자동 셋업
   ├─ data/  weights/  outputs/      # (비움; 코드로 채워짐)
   └─ notebooks/         # 00_, 01_ … 순서 접두사 노트북
```

- **각 세션은 독립 폴더**다. 자체 `requirements.txt`·`notebooks/`·`data/weights/outputs/`·README·conda 스크립트를 가진다.
- 세션 폴더는 **필요할 때 순차 추가**한다(session3~5는 미리 만들지 않는다).
- `PROJECT_ROOT` 는 항상 **현재 세션 루트**(예: `session2/`)를 가리킨다. 레포 루트가 아니다.

---

## 2. 불변 규칙 (프로젝트 헌법)

모든 세션·모든 단계에서 지킨다.

1. **경로**: 모든 경로는 **세션 루트(PROJECT_ROOT) 기준 상대경로**. 절대경로·개인 환경 경로 금지.
2. **데이터**: 레포에 이미지/대용량 데이터 파일을 포함하지 않는다. 노트북 **첫 셀에서 공개 출처로부터 코드로 다운로드**한다.
3. **버전 고정**: 핵심 라이브러리는 세션 `requirements.txt`에 버전을 고정하고 노트북 첫 셀에서 설치한다. **버전은 코랩에서 실제 확인된 값으로 고정하고 임의 추측하지 않는다.**
4. **셀 독립 실행**: 무료 Colab은 세션이 끊길 수 있다. 각 셀은 가능한 한 독립적으로, 중간 셀부터 다시 실행해도 동작하게 짠다(필요한 변수·로드는 셀 안에서 재확보).
5. **무료 GPU 한도**: 무료 Colab GPU·세션 시간 한도 안에서 끝나야 한다. 작은 모델 변형(tiny/small)·작은 이미지·적은 샘플로 구성한다.
6. **노트북 헤더**: 각 노트북 최상단에 마크다운으로 **"이 노트북에서 배우는 것 / 입력 / 출력"** 3줄을 둔다.
7. **환경 분기**: 로컬/코랩 차이(데이터 경로, 프로젝트 확보 방식)는 **한 곳(첫 셀)에서만** 분기하고, 이후 코드는 양쪽에서 동일하게 동작한다.
8. **역할 분리**: GPU가 필요한 작업은 **노트북**, 클릭하며 보는 열람용은 **HTML**. 한 산출물에 섞지 않는다.
9. **언어**: 설명·마크다운·주석은 **한국어**(대상이 한국 연구원). 코드 식별자는 영어 관례.
10. **파일 네이밍**: 노트북은 **순서 접두사**(`00_`, `01_`, `02_`…)를 붙인다.
11. **재현성**: 무작위성이 결과에 영향을 주는 곳은 **시드를 고정**한다.
12. **검증 우선**: 각 산출물은 **로컬과 코랩 양쪽에서 실제로 실행해 통과 조건을 확인**한 뒤 완료로 본다.

## 3. 환경 분기 원칙 (반드시 첫 셀 한 곳에서)

로컬/코랩 차이는 **노트북 첫 셀에서만** 처리하고, 그 외 코드는 양쪽에서 동일하게 동작한다(규칙 7).

- **코랩의 "프로젝트 확보"는 한 함수(`acquire_project()`)에 격리**한다. 배포 방식이 바뀌어도 이 함수와 README의 배지/링크만 교체하면 되게 한다.
- **기본 = 공개 GitHub 레포 클론.** 레포: `https://github.com/BSLJunhyeonJeon/AI_COP` (Public, 기본 브랜치 `main`).
  - "Open in Colab" 배지는 해당 세션의 `notebooks/00_setup.ipynb` 를 직접 연다.
  - 첫 셀에서 레포를 `/content/AI_COP` 로 클론한 뒤 **해당 세션 폴더(`/content/AI_COP/<session>`)로 `cd`** 한다. 그 위치가 `PROJECT_ROOT`. 공개 레포라 학생은 토큰·계정·인증이 전혀 필요 없다.
  - **멱등 처리**: `/content/AI_COP` 가 이미 있으면 다시 클론하지 말고 `cd`(또는 `git pull` 후 `cd`). clone 실패로 셀이 죽지 않게 방어한다.
- **로컬**: 레포를 클론한 폴더 안의 **세션 폴더가 작업 루트**다. `requirements.txt`(세션 폴더에만 존재)를 마커로 상위 탐색해 `PROJECT_ROOT`(세션 루트)를 잡고 `cd` 한다. 레포 루트에는 `requirements.txt` 가 없어야 한다.
- **로컬 환경 구성**: 각 세션의 `setup_env.bat`(윈도우)/`setup_env.sh`(맥·리눅스)로 conda 환경(`ai_cop_<session>`)을 만든다. Python·torch 버전은 **코랩과 일치**하도록 핀한다(코랩 값은 `00_setup.ipynb` 셀 5 출력에서 확인).

## 4. 작업 방식 (클로드 코드)

- **한 단계씩**: 현재 제공된 단계/지시 범위만 구현한다. 다음 단계로 임의로 넘어가지 않는다.
- **통과 조건 자가 점검**: 구현 후 해당 통과 조건을 스스로 점검하고 *무엇을 만들었는지 + 사람이 확인하는 방법*을 보고한다.
- **양쪽 실행 확인**: 가능한 범위에서 로컬·코랩 양쪽 동작을 확인하고, 불가하면 어디까지 확인했는지 명시한다.
- **요청 외 기능 금지**: 스펙에 없는 기능·화면·옵션을 임의로 추가하지 않는다. 필요해 보이면 추가하지 말고 **질문/메모로 남긴다**.
- **초보자 기준**: 모든 산출물은 코딩을 모르는 사용자가 받아서 **노트북을 열고 ▶만 눌러** 따라갈 수 있어야 한다.
