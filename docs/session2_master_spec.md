# 박사후연구원 학습공동체 2회차 — 구현 마스터 스펙

> 이 문서는 클로드 코드(Claude Code)가 2회차 수업 자료를 구현할 때 **항상 적용되는 기준**입니다.
> 단계별 작업 지시는 별도 스펙 문서로 **하나씩** 제공됩니다. 각 단계 스펙은 이 문서에 종속되며, 충돌 시 **이 문서가 우선**합니다.
> 한 번에 전체를 구현하지 말고, 제공되는 단계 순서대로 **하나씩** 진행하세요.
>
> **현행 기준 안내(멀티세션 마이그레이션 후)**: 이 문서는 session2 의 기획 기준(참고용/이력)이다. 범용 규칙의 최신·단일 출처는 레포 루트 [`CONVENTIONS.md`](../CONVENTIONS.md) 이고, 폴더 구조는 멀티세션(`session2/…`)으로 전환됐다. 아래 §4의 플랫 폴더 구조·경로 표기는 이력으로 남겨둔다(실제 경로는 `session2/` 기준).

## 1. 배경과 목표

- **대상**: 생명공학 분야 박사후연구원 3~4명. **AI/머신러닝 완전 초보**이며 파이썬·코딩 경험이 없다고 가정한다.
- **형식**: 오프라인 정기모임 1회, **약 2시간**.
- **2회차 학습 목표**: 다음 셋을 **개념과 체험으로** 이해시키는 것.
  1. 분류(classification)·감지(detection)·분할(segmentation) 세 태스크가 각각 무엇인지
  2. 각 태스크가 요구하는 데이터 형식이 무엇인지 (정밀할수록 라벨 비용이 큼)
  3. 데이터가 부족할 때 증강으로 어떻게 늘리는지
- **범위 한정**: 실제 전이학습(모델 훈련)은 **3회차**에서 다룬다. 2회차는 **사전학습 모델 추론 데모 수준**까지만 한다. 절대 2회차에서 훈련 루프를 돌리지 않는다.
- **설계 철학**: 코드 작성·환경 구축 설명에 시간을 쓰지 않는다. 연구원은 완성된 프로젝트를 받아 **노트북을 열고 ▶만 누른다**. 모든 자료는 "방금 들은 개념을 눈으로 바로 확인"하도록 시각적·체험적으로 만든다.

## 2. 최종 산출물

하나의 **프로젝트 루트**로 만든다. 구글 드라이브 폴더로 통째 배포하며, 각 노트북 상단에 "Open in Colab" 배지를 둔다. 구성:

- **실습용 Colab 노트북들** — GPU가 필요한 작업을 담당.
- **열람용 HTML 자료** — 개념 설명 + 증강 애니메이션 + 실습 결과 캡처/녹화. **claude.ai가 별도로 제작**하므로, 클로드 코드의 구현 범위는 *HTML에 들어갈 결과물(캡처·녹화·데이터·애니메이션)을 만들어 두는 것*까지다.
- **샘플 데이터** — 레포/폴더에 포함하지 않고 코드로 다운로드.

---

## 3. 불변 규칙 (프로젝트 헌법)

모든 단계에서 반드시 지킨다.

1. **경로**: 모든 경로는 프로젝트 루트 기준 **상대경로**. 절대경로·개인 환경 경로 금지.
2. **데이터**: 레포/폴더에 이미지 파일을 포함하지 않는다. 노트북 **첫 셀에서 공개 출처로부터 코드로 다운로드**한다.
3. **버전 고정**: 핵심 라이브러리는 `requirements.txt`에 버전을 고정하고, 노트북 첫 셀에서 설치한다. (Colab 기본 환경이 수시로 바뀌므로 재현성 확보용)
4. **셀 독립 실행**: 무료 Colab은 세션이 끊길 수 있다. 각 셀은 가능한 한 독립적으로, **중간 셀부터 다시 실행해도 동작**하도록 짠다. (필요한 변수·로드는 셀 안에서 재확보)
5. **무료 GPU 한도**: 무료 Colab GPU와 세션 시간 한도 안에서 끝나야 한다. **작은 모델 변형(tiny/small)·작은 이미지·적은 샘플**로 구성한다.
6. **노트북 헤더**: 각 노트북 최상단에 마크다운으로 **"이 노트북에서 배우는 것 / 입력 / 출력"** 3줄을 둔다.
7. **환경 분기**: 로컬/코랩 차이(데이터 경로, 드라이브 마운트 여부)는 **한 곳에서만** 분기 처리하고, 이후 코드는 양쪽에서 동일하게 동작한다.
8. **역할 분리**: GPU가 필요한 작업은 **노트북**, 클릭하며 보는 열람용은 **HTML**. 하나의 산출물에 섞지 않는다.
9. **언어**: 설명·마크다운·주석은 **한국어**(대상이 한국 연구원). 변수·함수 등 코드 식별자는 영어 관례를 따른다.
10. **파일 네이밍**: 노트북은 **순서 접두사**를 붙인다(`00_`, `01_`, `02_`…). 연구원이 번호대로 따라가도록.
11. **재현성**: 무작위성이 결과에 영향을 주는 곳은 **시드를 고정**한다.
12. **검증 우선**: 각 단계 산출물은 **로컬과 코랩 양쪽에서 실제로 실행해 통과 조건을 확인**한 뒤 완료로 본다.

## 4. 폴더 구조 (기준)

```
project_root/
├─ README.md                # 프로젝트 소개 + Colab 열기 안내
├─ requirements.txt         # 버전 고정
├─ data/                    # (비움; 첫 셀에서 다운로드되어 채워짐)
├─ weights/                 # 사전학습 가중치 (다운로드)
├─ outputs/                 # 실행 결과 저장 (캡처·마스크·애니메이션 등)
└─ notebooks/
   ├─ 00_setup.ipynb        # 환경 확인 + 데이터 다운로드
   ├─ 01_inference_demo.ipynb
   ├─ 02_labeling_sam2.ipynb
   └─ 03_augmentation.ipynb
```

파일명·구성은 단계 진행에 따라 조정될 수 있으나, **구조 원칙(데이터 분리·번호 접두사·outputs 저장)은 유지**한다.

## 5. 단계 목록과 통과 조건

순서대로 하나씩 진행한다. **각 단계는 통과 조건을 충족해야 다음으로 넘어간다.**

### 0단계 — 프로젝트 뼈대
- **내용**: 폴더 구조 생성, `requirements.txt` 버전 고정, 환경 분기(로컬/코랩) 한 곳 구현, "Open in Colab" 배지, 데이터 다운로드 셀(골격), `00_setup.ipynb`.
- **통과 조건**: 빈 셋업 노트북이 **로컬과 코랩 양쪽에서 에러 없이** 열리고 실행된다. GPU 사용 가능 여부를 출력한다. 상대경로 기준으로 프로젝트 루트가 올바로 잡힌다.

### 1단계 — 데이터 준비
- **내용**: 2회차용 공개 샘플 데이터 선정·다운로드(클래스 2~3개, 작은 이미지, 적은 수량). 분류·감지·분할 **각 형식 예시**를 보여줄 최소 데이터.
- **통과 조건**: 첫 셀 실행만으로 `data/`가 채워지고, 분류(폴더 분류)·감지(박스 라벨)·분할(마스크) **각 형식 예시가 한 눈에 확인**된다. 무료 GPU에 부담 없는 크기다.

### 2단계 — 실습1: 대표 모델 추론 데모
- **내용**: 세 태스크 각 대표 사전학습 모델로 추론 결과를 시각화. **일반 도메인 사전학습이라 생물 이미지에서 빗나가는 실패 사례를 1~2개 의도적으로 포함**(→ 3회차 전이학습 동기로 연결).
- **통과 조건**: 노트북 ▶ 실행만으로 분류(예측 라벨)·감지(박스)·분할(색칠) 결과가 이미지 위에 표시된다. 성공/실패 사례가 모두 보인다. 결과 이미지가 `outputs/`에 저장된다.

### 3단계 — 실습2: SAM2 라벨링 체험
- **내용**: SAM2로 분할 라벨링을 **클릭 기반으로 직접 체험**(분할에 집중). SAM2가 라이브에서 안 뜰 경우를 대비한 **백업 정적 결과**도 노트북에 포함.
- **통과 조건**: 작은 모델·작은 이미지로 무료 GPU에서 SAM2가 동작하고, 클릭 한 번에 마스크가 생성·저장된다. 백업 결과만으로도 흐름 시연이 가능하다.

### 4단계 — 증강
- **내용**: 어파인(기하) 변환과 색조/밝기(픽셀) 변환을 보여줌. **핵심: 이미지와 라벨(박스·마스크)이 함께 변형되는지를 시각적으로 구분** — 기하 변환은 라벨도 변하고, 픽셀 변환은 라벨 그대로. 노트북 실습용 + **claude.ai HTML에 넣을 애니메이션(또는 프레임 시퀀스) 생성 스크립트**.
- **통과 조건**: 노트북에서 원본→증강 결과와 라벨 변화가 나란히 보인다. 애니메이션/프레임이 생성되어 `outputs/`에 저장되고 HTML에 삽입 가능한 형태다. **"나쁜 증강"(도메인에 안 맞는 변환) 경고 예시**를 포함한다.

### 5단계 — HTML 열람 자료 (claude.ai 담당)
- **내용**: 0~4단계 결과(캡처·녹화·애니메이션)와 개념 설명(분류·감지·분할, 데이터 요구사항, 파이프라인 개요, 일반화/과적합, 증강 개념)을 묶은 인터랙티브 HTML.
- **통과 조건**: 브라우저에서 단독 열람 가능. 개념과 결과가 연결되어 보이고, GPU 의존 동작은 미리 만든 결과(이미지·영상)로 대체되어 있다.

## 6. 클로드 코드 작업 방식

- **한 단계씩**: 현재 제공된 단계 스펙만 구현한다. 다음 단계로 임의로 넘어가지 않는다.
- **통과 조건 자가 점검**: 구현 후, 해당 단계 통과 조건을 스스로 점검하고 결과를 보고한다 — *무엇을 만들었는지 + 사람이 확인하는 방법*.
- **양쪽 실행 확인**: 가능한 범위에서 로컬·코랩 양쪽 동작을 확인한다. 불가하면 어디까지 확인했는지 명시한다.
- **요청 외 기능 금지**: 스펙에 없는 기능·화면·옵션을 임의로 추가하지 않는다. 필요해 보이면 추가하지 말고 **질문/메모로 남긴다**.
- **초보자 기준**: 모든 산출물은 코딩을 모르는 사용자가 받아서 ▶만 눌러 따라갈 수 있어야 한다.

## 7. 열린 결정 (다음 단계 진행 전 확정 필요)

아래는 아직 정해지지 않은 선택지로, 단계 스펙을 쓰기 전에 확정한다.

1. **샘플 데이터 출처(1단계)**: 어떤 공개 데이터셋을 쓸지. 생명공학 친화적인 작은 공개 데이터(예: 세포/조직 이미지) 후보를 좁혀야 함.
2. **실습1 대표 모델(2단계)**: 태스크별 어떤 사전학습 모델로 데모할지. (감지 = YOLO11은 사실상 확정, 분류·분할 모델 선택 필요)
3. **실습2 범위(3단계)**: SAM2 분할 **하나만** 집중할지, 분류("폴더에 나눠 담기")를 가볍게 추가할지.
4. **합성 증강(4단계)**: copy-paste 등 합성을 **개념(HTML)으로만** 다룰지, 간단한 실습까지 넣을지.
