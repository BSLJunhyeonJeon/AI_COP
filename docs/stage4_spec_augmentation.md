> **범위**: **4단계(증강)만**. **모델 학습·추론 금지, SAM2 실행 금지(3단계 마스크 산출물 재사용만), HTML 조립 금지(5단계)** — 단 **HTML에 넣을 애니메이션 자산은 여기서 생성**한다. 스펙 외 기능 임의 추가 금지.
> **방식**: `session2/notebooks/04_augmentation.ipynb` 신규 + `session2/requirements.txt` 갱신(2단계 핀 확정 + albumentations 추가). 작업 후 커밋·push, 양쪽 확인 보고.

---

## 1. 목적

- 증강의 핵심 직관을 눈으로: **기하 변환(어파인·회전·플립)은 라벨(박스/마스크)도 함께 움직이고, 픽셀 변환(밝기·색조)은 라벨이 그대로**다.
- 증강은 *그럴듯한 변형*으로 데이터 다양성을 흉내 내는 것이지 **새 정보를 만드는 게 아니다.**
- **나쁜 증강 경고**: 도메인에 안 맞는 변환은 오히려 해롭다 — 예: 혈액 도말의 **염색 색이 진단 정보인데 색조를 극단적으로 바꾸면** 의미가 파괴된다.
- 5단계 HTML에 넣을 **애니메이션(또는 프레임 시퀀스)**을 생성한다.
- **학습/추론/SAM2 실행 없음.**

---

## 2. 도구·데이터 (확정)

- **albumentations** (신규 의존성) — 이미지 + 박스 + 마스크를 **함께** 변형하는 표준 라이브러리.
  - API 패턴(정확한 인자는 코랩 설치 버전(셀 2 출력)에 맞춰 확인):
    ```python
    import albumentations as A
    transform = A.Compose([...], bbox_params=A.BboxParams(format="pascal_voc", label_fields=["labels"]))
    t = transform(image=img, mask=mask, bboxes=boxes, labels=labels)
    # t["image"], t["mask"], t["bboxes"] 가 함께 변형됨
    ```
  - **기하(라벨도 변함)**: `Affine`/`HorizontalFlip`/`Rotate`/`ShiftScaleRotate` 등.
  - **픽셀(라벨 그대로)**: `RandomBrightnessContrast`/`HueSaturationValue`/`ColorJitter` 등.
- **데이터(재사용)**: 혈액 도말 1장 + VOC 박스(1단계 `data/detection`), 마스크 1장(1단계 Otsu 또는 3단계 SAM2 `clickmask_*.png`). 없으면 최소 확보(노트북 단독 실행).

## 4. `04_augmentation.ipynb` 구성 (셀)

헤더 3줄, 각 셀 독립 실행.

**셀 1 — 환경 셋업**: 00~03과 동일 패턴.

**셀 2 — 의존성 설치**: `requirements.txt` 설치(albumentations 포함). 설치된 albumentations 버전 출력(핀용).

**셀 3 — 입력 확보**: 혈액 도말 이미지 + 박스 + 마스크(재사용; 없으면 최소 확보).

**셀 4 — 기하 증강 (라벨도 변함)**: 어파인/회전/플립을 **image + boxes + mask에 함께** 적용. 원본 vs 증강을 나란히, **박스·마스크가 이미지를 따라 움직이는 것**을 보여준다. 캡션: "기하 변환 → 라벨도 같이 변한다." 그림 `outputs/` 저장.

**셀 5 — 픽셀 증강 (라벨 그대로)**: 밝기/대비/색조를 **이미지에만** 적용. **박스·마스크 좌표는 그대로**. 원본 vs 증강 나란히. 캡션: "픽셀 변환 → 라벨은 그대로." 그림 저장.

**셀 6 — 나쁜 증강 경고**: 도메인 부적합 변환 예시(예: **극단적 색조/채도 변경으로 염색 색 정보 파괴**) + 경고 캡션. "도메인에 안 맞는 증강은 오히려 모델을 망친다(특히 색이 정보인 생물 이미지)." 그림 저장.

**셀 7 — 애니메이션 (HTML용)**: 점진적 기하 변환(예: 회전 각도를 조금씩) + **마스크가 따라 움직이는 프레임 시퀀스**를 만들어 **GIF(또는 PNG 프레임)로 `outputs/` 저장**. HTML 삽입 가능한 형태. (임베드 방식은 5단계에서 결정 — 여기선 자산만 생성)

**셀 8 — 종합 / 검증**: 핵심 3줄 요약(**기하=라벨 이동 / 픽셀=라벨 유지 / 나쁜 증강=해로움**) + 합성은 **개념만 한 줄**(copy-paste로 새 예시를 만드는 방법도 있으나 검증 필요). `outputs/` 상태 출력.
