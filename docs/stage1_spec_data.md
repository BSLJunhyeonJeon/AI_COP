# 1단계 스펙 — 데이터 준비 (혈액세포 테마)

> **대상**: 클로드 코드.
> **전제**: `CONVENTIONS.md`(공통 규칙)와 0단계 결과(`session2/` 구조, `00_setup.ipynb`) 위에 쌓는다. 충돌 시 **CONVENTIONS.md 우선**.
> **범위**: **1단계(데이터 다운로드 + 세 형식 예시)만** 구현한다. 2단계 이후(사전학습 추론, YOLO/분할 모델, SAM2, 증강)는 **만들지 않는다.** 스펙 외 기능 임의 추가 금지.
> **방식**: `session2/notebooks/01_data.ipynb` 신규 작성 + `session2/requirements.txt`에 의존성 추가. 작업 후 커밋·push. 코랩·로컬 양쪽 실행을 확인하고 보고한다.

---

## 1. 목적

- 분류(classification)·감지(detection)·분할(segmentation) 세 태스크의 **데이터 형식 차이**를 코딩 모르는 연구원이 **눈으로** 이해하게 한다(개념 학습과 짝을 이루는 실습).
- 혈액세포 한 테마로 **가볍고 공개된 샘플**을 코드로 받아 `data/`에 형식별로 정리한다.
- **모델 학습·추론은 하지 않는다**(2단계 영역). 1단계는 순수 데이터 + 시각화.

핵심 메시지(연구원이 가져갈 것): *같은 종류의 이미지라도, 분류 → 감지 → 분할로 갈수록 라벨이 정밀해지고 만드는 비용이 커진다.*

---

## 2. 데이터 출처 (확정)

- **분류 — BloodMNIST (MedMNIST)**: `pip install medmnist` 로 받는다. 28/64/128/224 크기 옵션 중 **가시성을 위해 64px 권장**. 라이선스 CC BY 4.0. 8개 혈액세포 클래스 중 **시각적으로 구분되는 2~3개만 사용**(예: 림프구·단핵구·호중구 — 클래스명/인덱스는 medmnist의 INFO에서 가져옴).
- **감지 — BCCD (Blood Cell Count and Detection)**: GitHub `Shenggan/BCCD_Dataset`. 혈액 도말 이미지 + WBC/RBC/혈소판 바운딩 박스(**VOC XML 형식**), MIT 라이선스. **shallow clone 후 소수 이미지만** 사용한다(전체 364장 중 2~3장).
- **분할(형식 예시) — 가벼운 쪽으로**: 마스크가 *"원본과 같은 크기의 라벨 이미지(픽셀값 = 클래스)"*임을 보여줄 **1~2개 예시면 충분**하다. 다운로드 부담이 적은 방법을 택한다:
  - (선호) BBBC 핵 세트(예: BBBC038, CC0)에서 **image+mask 1~2쌍만** 받기. 전체 zip이 무겁거나 소수만 받기 어려우면 →
  - 혈액세포 이미지에 **간단 임계처리(예: Otsu)로 만든 예시 마스크**로 대체. (목적은 "마스크가 데이터로서 이렇게 생겼다"를 보여주는 것이므로 어느 쪽이든 무방)
  - **주의**: 실제 분할 라벨링은 **3단계에서 SAM2가 혈액 이미지에 직접** 수행한다. 1단계는 형식 소개만.

> 각 출처·라이선스(BloodMNIST CC BY 4.0 / BCCD MIT / BBBC CC0 등)를 노트북에 **주석으로 명시**한다.

## 3. 산출물

- `session2/notebooks/01_data.ipynb` (신규)
- `data/` 하위 형식별 정리:
  - `data/classification/<클래스명>/*.png` — "폴더 이름 = 클래스" 구조
  - `data/detection/` — 이미지 + 라벨(박스)
  - `data/segmentation/` — 이미지 + 마스크
- `session2/requirements.txt` 에 새 의존성 추가(버전 고정)

> 데이터 파일은 **레포에 커밋하지 않는다**(공통 규칙 2, `.gitignore`). 코드로 받고, `data/` 폴더는 `.gitkeep`만 유지.

## 4. `01_data.ipynb` 구성 (셀)

각 셀은 **독립 실행 가능**(공통 규칙 4), 맨 위 **헤더 3줄**(규칙 6).

**헤더 (마크다운)** — 배우는 것 / 입력 / 출력. (이 노트북은 "세 태스크 데이터가 어떻게 다른지 직접 보는 곳")

**셀 1 — 환경 셋업**: `00_setup.ipynb` 셀 1과 **동일한 패턴**(코랩이면 레포 클론→`session2`로 `cd`, 로컬이면 마커로 루트 탐색). 공통 로직을 작은 헬퍼로 빼도 되지만 **클론 이후에만 임포트**되게 한다.

**셀 2 — 의존성 설치**: `requirements.txt` 설치(이번에 추가된 `medmnist` 등 포함). 코랩 사전설치 패키지는 재설치하지 않는다.

**셀 3 — 분류 데이터 (BloodMNIST)**: 클래스 2~3개 선택 → 각 소수(예: 5장)를 PNG로 `data/classification/<클래스명>/` 에 저장. 출력으로 **"폴더 구조 = 라벨"**임을 보여준다(폴더 트리 + 샘플 몇 장).

**셀 4 — 감지 데이터 (BCCD)**: 소수 이미지 + VOC 박스를 `data/detection/` 에 저장. 출력으로 **박스 라벨 형식**을 보여준다 — 한 이미지의 박스 목록을 `클래스, xmin, ymin, xmax, ymax` 로 출력.

**셀 5 — 분할 형식 예시**: image+mask 1~2쌍을 `data/segmentation/` 에 저장. 출력으로 **마스크가 데이터로서 어떻게 생겼는지** 보여준다 — 마스크 배열의 shape와 "픽셀값이 곧 클래스"임을(예: 0=배경, 1=세포) 출력.

**셀 6 — 세 형식 비교 시각화 (이 노트북의 핵심)**: 한 그림에 나란히 —
  - 분류: 이미지 + **라벨 글자만**(공간 정보 없음)
  - 감지: 이미지 + **바운딩 박스**(+클래스)
  - 분할: 이미지 + **마스크 오버레이**(픽셀 단위)
  아래에 한 줄 설명: *"같은 종류 이미지, 왼쪽→오른쪽으로 라벨이 정밀해지고 비용이 커진다."* 이 그림을 `outputs/` 에 저장한다.

**셀 7 — 검증 / 요약**: `data/classification·detection·segmentation` 이 채워졌는지(형식별 개수) 출력하고, **배운 것 3줄 요약**(세 형식 + 라벨 비용 증가)을 출력한다.

---

## 5. 제약

- **가볍게**: BloodMNIST 소량(클래스당 몇 장), BCCD 2~3장, 분할 1~2 예시. 전체 다운로드를 작게 유지.
- **GPU 불필요**: 1단계는 데이터·시각화만이라 무료 Colab CPU로 충분하다(GPU 요구 금지).
- **셀 독립 실행**: 중간 셀부터 다시 실행해도 동작.
- **언어**: 설명·주석 한국어, 코드 식별자 영어.
- **버전 고정**: 새 의존성(`medmnist`, 시각화용 라이브러리 등)은 **코랩에서 확인된 버전으로** `requirements.txt`에 고정. 0단계에서 박은 `torch==2.11.0` 핀은 유지.
- **데이터 비커밋**: 받은 데이터·마스크·그림이 git에 추가되지 않게 한다(`.gitignore` 규칙 준수, `outputs/`의 비교 그림은 예외적으로 두려면 `.gitkeep`만 유지하고 그림 자체는 무시).
