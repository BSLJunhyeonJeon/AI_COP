> **대상**: 클로드 코드.
> **전제**: `CONVENTIONS.md`(공통 규칙) + 0·1단계 결과 위에 쌓는다. 충돌 시 **CONVENTIONS.md 우선**.
> **범위**: **2단계(사전학습 추론 데모)만** 구현한다. **모델 학습·전이학습·파인튜닝 금지(그건 다음 세션), SAM2 금지(3단계), 증강 금지(4단계).** 스펙 외 기능 임의 추가 금지.
> **방식**: `session2/notebooks/02_inference_demo.ipynb` 신규 + `session2/requirements.txt` 갱신(1단계 핀 확정 + 2단계 의존성 추가). 작업 후 커밋·push. 코랩·로컬 양쪽 실행 확인 후 보고.

---

## 1. 목적

- 분류·감지·분할 각 **대표 사전학습 모델**이 *추론*에서 무엇을 하는지 눈으로 본다.
- **핵심 메시지**: 일반 도메인(ImageNet/COCO/ADE20K)으로 학습된 모델은 **일반 사진에선 잘 맞지만(성공), 혈액세포에선 빗나간다(실패).** 이 대비가 이 실습의 전부다.
- 이 실패가 **다음 세션의 전이학습(너희 데이터로 모델 길들이기)의 동기**임을 명시적으로 연결한다.
- **학습/파인튜닝은 하지 않는다 — 추론만.**

---

## 2. 모델·입력 (확정)

세 태스크 모두 **추론 전용**, 작은 사전학습 모델:

- **분류 — torchvision ImageNet 분류기**(가벼운 것, 예: `resnet18` 또는 `mobilenet_v3_small`). 1000개 일반 클래스. 가중치에 딸린 전처리/카테고리를 사용.
- **감지 — Ultralytics YOLO11n**(`yolo11n.pt`, COCO 80클래스 사전학습). *커리큘럼의 YOLO11과 연결.*
- **분할 — HuggingFace SegFormer-b0**(예: `nvidia/segformer-b0-finetuned-ade-512-512`, ADE20K 150클래스 시맨틱). *커리큘럼의 SegFormer와 연결.*

**입력 이미지 두 종류** (각 태스크에서 둘 다 돌려 대비):

- **성공 입력**: 일반 장면 사진 1장(거리/사람/차량 등이 있는 **안정적인 공개·퍼블릭도메인 이미지** — 예: ultralytics 샘플 또는 위키미디어 퍼블릭도메인). 세 모델이 제 도메인이라 합리적으로 맞힌다. (다운로드 URL은 코랩에서 실제로 받히는지 확인할 것)
- **실패 입력**: 혈액 이미지 — 분류/분할은 **BloodMNIST 세포 1장**, 감지는 **BCCD 도말 1장**. (1단계와 동일 출처, **최소량으로 노트북에서 직접 확보** — 노트북이 단독 실행되게)

> **SAM/SAM2는 여기서 쓰지 않는다**(3단계 라벨링 전용). 분할 실패 데모는 **클래스 기반 시맨틱 모델(SegFormer-ADE)**로 한다 — 그래야 "혈액 이미지를 엉뚱한 ADE 클래스로 칠한다"는 실패가 드러난다. (SAM 류는 클래스 없이 덩어리를 그럴듯하게 잡아 실패가 안 보임)

## 4. `02_inference_demo.ipynb` 구성 (셀)

헤더 3줄, 각 셀 독립 실행(공통 규칙 4·6).

**셀 1 — 환경 셋업**: 00/01과 **동일 패턴**(코랩 클론→`session2` cd, 로컬 마커 탐색).

**셀 2 — 의존성 설치**: `requirements.txt` 설치(추가된 torchvision/ultralytics/transformers 포함). 코랩 사전설치본 재설치 금지. **설치된 새 의존성 버전 출력**(핀용: torchvision/ultralytics/transformers).

**셀 3 — 입력 확보**: 성공용 일반 이미지 1장 + 실패용 혈액 이미지(분류/분할용 BloodMNIST 세포 1장, 감지용 BCCD 도말 1장)를 셀에서 직접 받는다(최소량).

**셀 4 — 분류 데모 (ImageNet 분류기)**: 성공 이미지 → top-3 라벨, 혈액 세포 → top-3 라벨을 **나란히** 표시. 혈액 쪽은 혈액세포 클래스가 1000개 안에 없어 **엉뚱한 라벨**이 나옴을 보여준다. 그림 `outputs/` 저장.

**셀 5 — 감지 데모 (YOLO11n / COCO)**: 성공 이미지 → 박스(사람·차 등), BCCD 도말 → **박스가 거의 없거나 엉뚱**함을 나란히 표시. COCO 80클래스에 혈액세포가 없음을 보여준다. 그림 저장.

**셀 6 — 분할 데모 (SegFormer-b0 / ADE)**: 성공 이미지 → 의미있는 세그먼트(도로·차량·사람 등), 혈액 이미지 → **ADE 클래스로 엉뚱하게 칠함**을 나란히 표시. 그림 저장.

**셀 7 — 종합 + 다리 (검증 포함)**: "세 모델 모두 일반 사진엔 OK, 혈액엔 빗나감 — **너희 도메인을 배운 적이 없어서**. 다음 세션에 바로 이 모델들(YOLO11·SegFormer)을 **너희 라벨로 전이학습**하면 맞히게 된다." 종합 그림을 `outputs/`에 저장하고, `outputs/` 상태 + 한 줄 요약을 출력한다.

## 6. 제약

- **학습/파인튜닝 금지** — 추론만(`eval()`/`no_grad`). 전이학습은 다음 세션.
- **SAM2 금지**(3단계), **증강 금지**(4단계).
- **무료 Colab GPU·세션 한도 안.** GPU 있으면 사용, 없으면 CPU 폴백(모델이 작아 가능). 사전학습 가중치는 각 라이브러리 기본 캐시로 자동 다운로드(작음) — `weights/` 사용은 선택.
- **셀 독립 실행**, 설명·주석 **한국어**. 그림 안 글자는 폰트 호환을 위해 영문 가능(1단계처럼).
- **데이터·가중치·결과 비커밋**(`.gitignore` 준수).
- `ultralytics` 설치가 **기존 torch를 덮어쓰지 않게**(이미 만족하면 그대로 둠).
