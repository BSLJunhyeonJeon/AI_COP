## 1. 목적

- **SAM2로 클릭(포인트) 한 번에 세포 마스크가 생기는 걸 체험** → "분할 라벨(마스크)을 이렇게 만든다."
- **2단계와의 대비**: 일반 시맨틱 모델(SegFormer-ADE)은 혈액에서 실패했지만, 파운데이션 모델 **SAM2는 학습 없이(zero-shot) 클릭만으로 세포를 잘 따낸다.** 단, **클래스(이름)는 사람이 지정**한다(SAM2는 "어디"만 알고 "무엇"인지는 모름).
- 이렇게 만든 라벨이 **다음 세션 전이학습의 입력**이 된다.
- **학습/파인튜닝 없음** — SAM2 프롬프트 추론만.

## 2. 도구·데이터 (확정)

- **SAM2 = ultralytics의 `SAM`** (2단계에서 이미 설치됨 → **새 pip 의존성 없음**). `ultralytics>=8.2.70` 필요(이미 충족).
  - **작은 변형 사용**: `SAM("sam2_t.pt")` (또는 `sam2.1_t.pt`) — 코랩에서 실제 받히는 tiny 변형을 쓴다(무료 GPU 부담 최소).
  - **포인트 프롬프트 API(확인됨)**:
    ```python
    from ultralytics import SAM
    model = SAM("sam2_t.pt")
    results = model(image, points=[[x, y]], labels=[1])   # 양성 클릭=1, 음성=0
    mask = results[0].masks                                # 생성된 마스크
    ```
- **데이터**: 혈액 도말(BCCD) 1장 — 세포가 여러 개라 라벨링 체험에 적합. 1단계 `data/detection` 재사용, 없으면 BCCD 1장 확보(노트북 단독 실행 보장).
- **백업(라이브 안정성)**: SAM2 로드/실행 실패 대비 — 실패 시 **임시 마스크(예: 1단계와 동일한 Otsu)로 흐름을 시연**할 수 있게 한다. (라이브에서 SAM2가 안 떠도 수업이 안 깨지게)

## 4. `03_labeling_sam2.ipynb` 구성 (셀)

헤더 3줄, 각 셀 독립 실행.

**셀 1 — 환경 셋업**: 00/01/02와 동일 패턴.

**셀 2 — 의존성 확인**: `requirements.txt` 설치(SAM2용 새 deps 없음 — `ultralytics` 버전이 8.2.70 이상인지 확인 출력).

**셀 3 — 라벨링할 이미지 확보**: 혈액 도말 1장(1단계 `data/detection` 재사용, 없으면 BCCD 1장).

**셀 4 — SAM2 클릭 → 마스크 (핵심)**: 작은 SAM2 로드(GPU 있으면 사용). 세포 위 **점 1개**(예: WBC 중심 좌표)로 마스크 생성 후 오버레이. 예시 점 **2~3개**(예: WBC 중심, RBC 하나)로 "클릭마다 세포 하나가 잡힌다"를 보여준다. 생성 마스크를 `data/segmentation/` 저장. **백업**: SAM2 실패 시 Otsu 임시 마스크 + 안내 메시지로 흐름 유지.

**셀 5 — 라벨링 워크플로 + 대비**: "**클릭=마스크(SAM2) + 클래스=사람 지정 → 라벨 1개 완성**"을 보여준다. 손으로 폴리곤을 일일이 그리는 것 대비 **클릭 한 번이면 된다**는 점을 한 줄로. *(체험 팁)* 점 좌표를 바꿔 다시 실행하면 다른 세포를 "클릭"하는 셈 — 이 방식으로 체험한다. *(참고)* 실제 라벨링 도구로는 **CVAT + SAM 통합**이 있음(커리큘럼). 비교 그림 `outputs/` 저장.

**셀 6 — 종합 + 다리(검증 포함)**: "SAM2 클릭으로 분할 라벨을 쉽게 만든다(클래스는 사람이 붙임). 이렇게 만든 라벨로 **다음 세션에 모델을 전이학습**한다." `outputs/`·`data/segmentation/` 상태와 한 줄 요약 출력.

## 5. 제약

- **학습/파인튜닝 금지** — SAM2 프롬프트 추론만. **증강 금지**(4단계), **HTML 금지**(5단계).
- **무료 Colab GPU·세션 한도 안.** tiny SAM2 + 작은/적은 이미지. GPU 있으면 사용, 없으면 CPU(느리지만 가능 — 이미지 인코딩 1회 후 클릭은 빠름).
- **셀 독립 실행**, 설명·주석 **한국어**. 그림 안 글자는 폰트 호환 위해 영문 가능.
- **데이터·가중치·결과 비커밋**(`.gitignore`). 백업은 가능하면 **비커밋 방식(Otsu)** 선호.
- **진짜 클릭 캡처(ipywidgets 등)는 코랩에서 불안정** → 요구하지 않는다. **좌표 점 + "좌표 바꿔 재실행"** 방식으로 체험. (위젯을 넣더라도 선택적이고, 실패해도 노트북이 깨지지 않게)
